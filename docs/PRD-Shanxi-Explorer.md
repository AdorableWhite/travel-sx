# 山西全域智慧旅游平台（晋游记 / Shanxi Explorer）PRD

文档版本：V1.0  
适用平台：Web (PC) + H5/小程序/App (Mobile)

## 一、项目建设目的

1. 文化赋能与传播：通过数字化手段（VR、语音讲解、深度文章）将历史知识转化为旅游体验，解决“看不懂”的问题。  
2. 流量转化与留存：承接 IP 热度，将热门景点引流至高价值冷门点位。  
3. 提升出行效率：优化交通线路规划，提供一站式出行解决方案。  
4. 数字化服务升级：打通 PC 端与移动端，实现全场景覆盖。

## 二、痛点分析与解决方案

| 用户角色 | 核心痛点 | 解决方案 |
| --- | --- | --- |
| 游客 | 景点分散、交通接驳难查 | 智能行程规划，包车/拼车/公交接驳方案 |
| 游客 | 古建看不懂，缺乏深度讲解 | AI 导游 + 专业古建知识库 |
| 游客 | 饮食习惯差异，不知哪里正宗 | 面食地图 + 老字号认证推荐 |
| 游客 | 攻略碎片化，信息杂 | 官方精选路线（如“重走悟空路”） |
| 景区/商家 | 冷门景点无人问津 | 联票系统，热门带冷门 |
| 景区/商家 | 缺乏数字化营销渠道 | 商家入驻后台，发布实时优惠 |
| 文旅部门 | 缺乏统一数据看板 | 大数据中心，实时监控客流 |
| 文旅部门 | 二销能力弱 | 集成文创商城 |

## 三、功能模块划分

### 1. 核心业务模块

- **三晋览胜（景点）**：石窟、古建、山水、红色旅游分类展示，支持 VR 全景预览。  
- **晋善晋美（美食）**：山西面食大全、陈醋文化、特产推荐。  
- **古韵探秘（文化/攻略）**：历史时间轴、古建筑结构解析、壁画鉴赏、UGC 游记。  
- **智游山西（工具/预订）**：门票预订、酒店民宿、智能路线规划、租车服务。  
- **晋礼文创（商城）**：文创产品、土特产在线购买。  

### 2. 用户中心模块

我的行程、收藏夹、订单管理、足迹地图。

## 四、详细需求（PRD）

### 4.1 总体架构与交互差异

- **PC 端**：大图与时间轴动画，强调沉浸式浏览与复杂行程编辑。  
- **移动端**：卡片式设计，强调 LBS 推荐、扫码听讲解、一键导航。  

### 4.2 模块需求

#### 模块一：首页 (Home)

- **Banner**：四季风光或热门活动轮播。  
- **快捷入口**：  
  - Mobile：景点预约、线路推荐、美食地图、文创商城、交通指南  
  - PC：横向导航栏，悬停显示二级菜单  
- **热门推荐**：  
  - “悟空”巡礼（游戏取景地现实攻略）  
  - 当季必玩（根据天气推荐）  
- **搜索**：模糊搜索，景点/攻略/酒店联动展示。  

#### 模块二：景点详情 (Attractions)

- **基础信息**：名称、等级、开放时间、门票价格。  
- **位置服务**：高德/百度地图嵌入，移动端一键导航。  
- **深度内容**：  
  - 语音导览（普通话/山西话）  
  - 古建解构（3D 模型或 SVG 图解）  
- **周边配套**：5km 内厕所、停车场、餐厅。  
- **预约通道**：对接票务系统，生成二维码入园。  

#### 模块三：智能行程规划 (Smart Itinerary)

- **PC 端行程画布**：  
  - 拖拽景点到时间轴  
  - 计算景点间车程，时间不足预警  
  - 导出 PDF 行程单  
- **移动端随身路书**：  
  - 同步 PC 行程  
  - 进度提醒与下一站提示  
  - 一键求助（纠纷/紧急电话）  

#### 模块四：山西面食与美食 (Culinary)

- **面食百科**：刀削面、剔尖、莜面、栲栳栳等制作工艺与视频。  
- **探店地图**：LBS 推荐高评分面馆，支持图文评价。  
- **特产直邮**：陈醋、平遥牛肉等链接跳转商城。  

#### 模块五：社区与互动 (Community)

- **游记分享**：PC 富文本编辑器 / Mobile 多图上传。  
- **话题圈子**：#寻找最美古建#、#山西下雪了# 等讨论。  
- **打卡系统**：GPS 判定点亮徽章，兑换文创奖励。  

### 4.3 非功能性需求 (NFR)

- **性能**：首页加载 Mobile < 2s，PC < 3s。  
- **并发**：节假日高峰流量保障（票务接口重点）。  
- **兼容**：PC（Chrome/Firefox/Edge/Safari 新版），Mobile（iOS 12+/Android 8.0+/小程序）。  
- **安全**：身份证号、手机号 AES-256 加密存储；防 SQL 注入与 XSS。  
- **UI/UX**：砖红、土黄、墨黑为主色，金色点缀；标题书法感字体，正文无衬线字体。  

## 五、技术栈推荐

- **前端（Web/PC）**：React 或 Vue（建议 Nuxt 用于 SEO）。  
- **前端（Mobile）**：Uni-app 或 Taro。  
- **后端**：Spring Boot 或 Django/FastAPI。  
- **数据库**：MySQL + Redis + MongoDB。  
- **地图**：高德或百度地图 API。  
- **云服务**：对象存储 OSS + CDN。  

## 六、项目开发阶段规划（Roadmap）

1. **第一阶段（MVP，1.5 个月）**：首页、景点详情、基础搜索；小程序上线；覆盖 IP 相关景点。  
2. **第二阶段（完善，1.5 个月）**：用户系统、评论功能、地图导航接入；PC 端上线。  
3. **第三阶段（商业化与智能，2 个月）**：票务/酒店接入、商城模块、智能行程规划算法。
